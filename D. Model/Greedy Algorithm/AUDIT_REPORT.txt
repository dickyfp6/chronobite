================================================================================
7-PHASE EXECUTION FLOW (SINGLE SOURCE OF TRUTH)
================================================================================

Input: User Profile (age, weight, height, gender, activity, disease)
  ↓
Phase 1: NutritionService.calculate_nutrition_needs()
  → Returns: TDEE, nutrition constraints, food database
  ↓
Phase 2: GreedyAlgorithmInterface.initialize(food_db, constraints)
  → Prepares optimizer with data
  ↓
Phase 3: GreedyOptimizer.generate_menu()
  → For each meal: generates 3 candidates per course (per-100g)
  → Diversity check: ingredient similarity exclusion
  → Returns: MenuPlan with candidates (per-100g basis)
  ↓
Phase 4: GreedyOptimizer.optimize_portion()
  → Formula: portion_g = (target_kcal / kcal_per_100g) * 100
  → Clamps to PORTION_RANGE constraints
  ↓
Phase 5: GreedyOptimizer.scale_nutrients()
  → Formula: actual_nutrient = nutrient_per_100g * (portion_g / 100)
  → Scales: energy_kcal, protein_g, carbohydrate_g, fat_g
  → Returns: MenuPlan with scaled portions and nutrients
  ↓
Phase 6: Output MenuPlan to user
  → Full menu with all courses, portions, scaled nutrients
  ✓ Ready for UI display
  ↓
Phase 7: (Optional) PortionRebalancer.rebalance_menu()
  → If user makes substitutions (e.g., selects different drink)
  → Adjusts remaining portions to recover nutritional deficits
  → Does NOT change which foods are selected
  → Returns: Updated MenuPlan with rebalanced portions
